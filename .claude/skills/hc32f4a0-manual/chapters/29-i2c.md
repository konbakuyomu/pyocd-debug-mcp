# I2C — 集成电路总线

> ✅ 本知识卡已填充。最后更新：2026-03-25

## 模块概述

HC32F4A0 内置 6 个独立 I2C 单元（I2C1~I2C6），支持 I2C 总线和 SMBus（Ver.2.0）两种工作模式。提供主机/从机自动切换、多主机仲裁、标准模式（100 kbps）和快速模式（400 kbps）。每单元可配置 2 个独立从机地址（7 位/10 位格式），支持广播地址、SMBus 主机/默认/报警响应地址检测。内建双缓冲收发结构（DTR→DSR 发送、DSR→DRR 接收）、SCL 握手（时钟拉低等待）、SCL 电平超时检测、可编程数字滤波器（1~4 级）+ 模拟滤波器。通信错误/事件中断和 DMA 请求各 4 路。

## 关键特性

- 6 个独立单元，主机/从机模式自动切换，多主机仲裁
- **标准模式** 100 kbps，**快速模式** 400 kbps
- 2 个从机地址寄存器（SLR0/SLR1），各支持 7 位或 10 位地址格式
- SMBus 支持：主机地址、默认地址、报警响应地址检测；SCL 电平超时；PEC（需 CPU CRC）
- 双缓冲收发：DTR→DSR（发送）、DSR→DRR（接收），支持连续传输
- SCL 握手：从机发送/接收时自动拉低 SCL 等待软件处理
- 快速 ACK/NACK（FACKEN）：第 8 时钟下降沿保持 SCL 低电平，用于 PEC 校验
- BUSWAIT 模式：DRR 满时自动保持 SCL 低电平直到软件读取
- 数字滤波器（DNF[1:0]）：1~4 个 I2C 基准时钟周期滤波能力 + 模拟滤波器
- 软件复位：通信复位（PE=0, SWRST=1）/ 内部状态复位（PE=1, SWRST=1）
- 4 路中断：EEI（通信错误/事件 11 种）、RXI（接收满）、TXI（发送空）、TEI（发送结束）
- 波特率公式（DNFE=0, FREQ=000）：B = 1/[(SHIGHW+3+SLOWW+3)/ΦI2C + tr + tf]
- I2C 时钟源为 PCLK3

## 典型初始化流程

```c
/* === I2C Master 轮询读写（400kHz Fast Mode） === */
// 1. 使能时钟
FCG_Fcg1PeriphClockCmd(FCG1_PERIPH_I2C1, ENABLE);

// 2. 配置 GPIO 复用（SCL/SDA）
GPIO_SetFunc(GPIO_PORT_D, GPIO_PIN_03, GPIO_FUNC_49); // I2C1_SCL
GPIO_SetFunc(GPIO_PORT_F, GPIO_PIN_10, GPIO_FUNC_48); // I2C1_SDA

// 3. 初始化 I2C
stc_i2c_init_t stcI2cInit;
I2C_StructInit(&stcI2cInit);
stcI2cInit.u32ClockDiv = I2C_CLK_DIV4;
stcI2cInit.u32Baudrate = 400000UL;
stcI2cInit.u32SclTime  = 3UL;
I2C_DeInit(CM_I2C1);
I2C_Init(CM_I2C1, &stcI2cInit, NULL);  // NULL=不关心实际误差

// 4. 主机使能握手等待
I2C_BusWaitCmd(CM_I2C1, ENABLE);

// 5. 使能 I2C + 软复位
I2C_Cmd(CM_I2C1, ENABLE);
I2C_SWResetCmd(CM_I2C1, ENABLE);
I2C_SWResetCmd(CM_I2C1, DISABLE);

// 6. Master 发送
I2C_Start(CM_I2C1);
I2C_TransAddr(CM_I2C1, SLAVE_ADDR, I2C_DIR_TX);
I2C_TransData(CM_I2C1, txBuf, txLen);
I2C_Stop(CM_I2C1);

// 7. Master 接收
I2C_Start(CM_I2C1);
I2C_TransAddr(CM_I2C1, SLAVE_ADDR, I2C_DIR_RX);
I2C_MasterReceiveDataAndStop(CM_I2C1, rxBuf, rxLen);
```

## 常见陷阱与注意事项

1. ⚠️ **初始化必须先复位再配置**：标准流程为 PE=0→SWRST=1→PE=1→配置寄存器→SWRST=0，或使用 DDL 的 `I2C_DeInit()` + `I2C_Init()`
2. ⚠️ **时钟源为 PCLK3**：6 个 I2C 单元共享 PCLK3（APB3 总线），修改系统时钟需重新计算波特率
3. ⚠️ **通信前必须软复位**：Master 每次通信前建议执行 `I2C_SWResetCmd(ENABLE)` → `I2C_SWResetCmd(DISABLE)` 确保状态机干净
4. ⚠️ **RFULLF 不读 DRR 会导致 SCL 拉低**：接收满后不及时读取 DRR，硬件自动在下次 RFULLF 置位前的 SCL 时钟保持低电平（握手机制）
5. ⚠️ **DMA 收发边界处理**：DMA 发送需手动发第 1 字节；DMA 接收需手动处理最后 2 字节（倒数第 2 字节发 NACK + 产生 STOP 后读最后字节）
6. ⚠️ **EEI 中断优先级必须最高**：使用中断模式时，EEI（通信错误/事件）中断优先级必须高于 RXI/TXI/TEI
7. ⚠️ **从机发送结束必须读 DRR**：从机发送完成后（TENDF=1 或 NACKF=1），必须读一次 I2C_DRR 来释放 SCL 线
8. ⚠️ **SMBus 需设置 CMOS 输入电平**：使用 SMBus 时需通过 GPIO PCRxy.CINSEL 将 SCL/SDA 输入电平设为 CMOS 电平
9. ⚠️ **SHIGHW/SLOWW 波特率计算含固定偏移**：公式中有 +2 或 +3 的固定偏移量，取决于 DNFE 和 FREQ 设置
10. ⚠️ **10 位地址从机发送需两次地址匹配**：10 位寻址模式发送时，主机先发写地址头+低 8 位，再发重新开始+读地址头，从机需等两次 ADDR0 匹配
