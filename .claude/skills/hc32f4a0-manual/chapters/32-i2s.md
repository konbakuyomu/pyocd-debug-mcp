# I2S — 集成电路内置音频总线

> ✅ 本知识卡已填充。最后更新：2026-03-25

## 模块概述

I2S（Inter-IC Sound Bus）是专用于音频设备之间数据传输的串行总线模块。HC32F4A0 内置 4 个独立 I2S 单元（I2S1~I2S4），支持全双工/半双工通信，主/从模式，4 种音频协议标准（Philips、MSB 对齐、LSB 对齐、PCM），带 4 字深 × 32 位宽的收发 FIFO 缓冲区。

## 关键特性

- 4 个独立 I2S 单元（I2S1~I2S4）
- 通信方式：半双工（SD 引脚收或发）、全双工（SD 发送 + SDIN 接收）
- 主/从模式：主模式产生 CK/WS 时钟，从模式接收外部时钟
- 音频协议：I2S Philips / MSB 对齐 / LSB 对齐 / PCM（短帧/长帧同步）
- 数据格式：16/24/32 位数据，16/32 位通道帧宽度，MSB 先发
- 采样率支持：8k/16k/22.05k/32k/44.1k/48k/96k/192kHz
- 时钟源：内部 I2SCLK（PLLA/PLLH 的 P/Q/R 输出）或外部 EXCK 引脚
- 8 位线性预分频器（I2SDIV），ODD 位支持奇数分频
- 驱动时钟输出（MCK）= 256 × Fs
- 4 字深 FIFO + 阈值报警中断（TXBIRQWL/RXBIRQWL 可配置 0~4）
- 中断：发送缓冲报警、接收缓冲报警、接收上溢、发送下溢/上溢
- 6 根信号线：CK（时钟）、WS（字选择）、SD（数据）、SDIN（全双工输入）、EXCK（外部时钟）、MCK（驱动时钟）

## 寄存器速查

> I2S1: 0x4001E000 | I2S2: 0x4001E400 | I2S3: 0x40022000 | I2S4: 0x40022400

| 寄存器 | 偏移 | 用途 | 关键位字段 |
|--------|------|------|-----------|
| I2S_CTRL | 0x000 | 控制寄存器 | SRST, CLKSEL, DUPLEX, CKOE, LRCKOE, SDOE, MCKOE, ODD, WMS, EIE, RXIE, RXE, TXIE, TXE, TXBIRQWL[2:0], RXBIRQWL[2:0] |
| I2S_SR | 0x004 | 状态寄存器 | RXBF, RXBE, TXBF, TXBE, RXBA, TXBA |
| I2S_ER | 0x008 | 错误状态 | RXERR(接收上溢), TXERR(发送下溢/上溢)，写 1 清除 |
| I2S_CFGR | 0x00C | 配置寄存器 | PCMSYNC, CHLEN, DATLEN[1:0], I2SSTD[1:0] |
| I2S_TXBUF | 0x010 | 发送 FIFO | TXBUF[31:0] |
| I2S_RXBUF | 0x014 | 接收 FIFO | RXBUF[31:0]（只读） |
| I2S_PR | 0x018 | 预分频 | I2SDIV[7:0]（0/1 为禁用值，实际分频 = I2SDIV×2+ODD） |
| CMU_I2SCKSEL | CMU+0x12 | 时钟源选择 | I2S1~4CKSEL（PCLK/PLLHQ/PLLHR/PLLAP/PLLAQ/PLLAR） |

## 典型初始化流程

```c
/* 以 I2S1 主模式 Philips 标准 32kHz 播放为例 */

/* 1. GPIO 配置 */
GPIO_SetFunc(GPIO_PORT_F, GPIO_PIN_06, GPIO_FUNC_56);  /* I2S1_CK */
GPIO_SetFunc(GPIO_PORT_F, GPIO_PIN_08, GPIO_FUNC_56);  /* I2S1_WS */
GPIO_SetFunc(GPIO_PORT_F, GPIO_PIN_07, GPIO_FUNC_56);  /* I2S1_SD */
GPIO_SetFunc(GPIO_PORT_A, GPIO_PIN_05, GPIO_FUNC_56);  /* I2S1_MCK */

/* 2. 时钟源选择（PLL 输出作为 I2S 时钟基准） */
CLK_SetI2SClockSrc(CLK_I2S1, CLK_PERIPHCLK_PCLK);
FCG_Fcg1PeriphClockCmd(FCG1_PERIPH_I2S1, ENABLE);

/* 3. I2S 初始化 */
stc_i2s_init_t stcI2sInit;
stcI2sInit.u32ClockSrc    = I2S_CLK_SRC_PLL;
stcI2sInit.u32Mode        = I2S_MD_MASTER;
stcI2sInit.u32Protocol    = I2S_PROTOCOL_PHILLIPS;
stcI2sInit.u32TransMode   = I2S_TRANS_MD_HALF_DUPLEX_TX;
stcI2sInit.u32AudioFreq   = I2S_AUDIO_FREQ_32K;
stcI2sInit.u32ChWidth     = I2S_CH_LEN_16BIT;
stcI2sInit.u32DataWidth   = I2S_DATA_LEN_16BIT;
stcI2sInit.u32MCKOutput   = I2S_MCK_OUTPUT_ENABLE;
I2S_Init(CM_I2S1, &stcI2sInit);

/* 4. 开始发送 */
I2S_FuncCmd(CM_I2S1, I2S_FUNC_TX, ENABLE);
/* 轮询或 DMA 方式写入 I2S_TXBUF */
```

## 常见陷阱与注意事项

1. **I2SDIV 禁用值**：I2SDIV=0 和 I2SDIV=1 为禁用值，实际分频 = I2SDIV×2+ODD，最小有效分频为 4（I2SDIV=2, ODD=0）
2. **ODD 位设置顺序**：若要设 ODD=1，必须先设 I2S_PR 寄存器，再设 I2S_CTRL 寄存器
3. **主机暂停发送**：TXBUF 空后 I2S 自动暂停（停止时钟），TXERR 置位；不要直接清 TXE 关闭（会导致从机状态混乱），应等暂停后再操作
4. **主机只接收暂停**：需预写 2 帧 dummy 数据，提前 4~5 个数据 TXE 置 1 触发暂停序列
5. **从机启动时序**：Philips/MSB/LSB 模式下，WS 必须处于右声道电平状态时才能启动；PCM 模式下 WS 须为低电平
6. **从机最后一帧数据延迟**：从机接收时，最后一帧数据需等下次通信开始后才能读取
7. **发送中断写数据**：TXIRQOUT 中断中每次写 2 个数据时，必须先关 TXIE 再写再开，或每次只写 1 个
8. **32 位帧中 16/24 位数据**：低位自动补零，无需软件处理，只需单次读/写操作
9. **时钟源两层配置**：CMU_I2SCKSEL 选外设时钟来源，I2S_CTRL.CLKSEL 选分频基准（内部 PLL vs 外部 EXCK），两者独立
10. **只支持 32 位写寄存器**：所有 I2S 寄存器只能以 32 位宽写入
